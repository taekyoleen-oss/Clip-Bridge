# 📖 ClipBridge 사용 가이드

ClipBridge를 실제로 사용하는 방법을 단계별로 안내합니다.

## 🚀 시작하기

### 1. 개발 서버 실행

터미널에서 다음 명령어를 실행하세요:

```bash
pnpm dev
```

서버가 시작되면 다음과 같은 메시지가 표시됩니다:
```
  ▲ Next.js 14.2.0
  - Local:        http://localhost:3000
```

### 2. 브라우저에서 앱 열기

브라우저에서 [http://localhost:3000](http://localhost:3000)을 열어주세요.

## 💻 PC에서 사용하기

### 첫 실행 시 설정

1. **클립보드 권한 허용**
   - 브라우저에서 클립보드 읽기 권한을 요청하면 **"허용"** 클릭
   - Chrome/Edge: 주소창 왼쪽의 자물쇠 아이콘 클릭 > 권한 허용
   - Firefox: 설정 > 개인정보 및 보안 > 권한 > 클립보드

2. **권한 확인**
   - 앱 상단에 "클립보드 읽기 권한이 허용되었습니다" 메시지가 표시되면 성공

### 텍스트 복사 및 저장

1. **텍스트 복사**
   - 원하는 텍스트를 선택하고 `Ctrl+C` (또는 우클릭 > 복사)

2. **저장 대기 알림 확인**
   - 우측 하단에 Toast 알림이 나타남
   - 복사한 텍스트 미리보기 표시
   - 10초 카운트다운 시작

3. **저장 옵션 선택**
   - **"즉시 저장"**: 카운트다운 없이 바로 저장
   - **"취소"**: 저장하지 않고 알림 닫기
   - **자동 저장**: 10초 후 자동으로 저장

4. **저장 확인**
   - 저장된 클립이 화면 상단에 카드 형태로 표시됨
   - 최신 클립이 맨 위에 표시됨

### 저장된 클립 사용하기

1. **복사하기**
   - 클립 카드를 클릭하면 클립보드에 복사됨
   - 또는 복사 버튼(📋) 클릭

2. **공유하기**
   - 공유 버튼(📤) 클릭
   - 카카오톡, 문자 등으로 공유 가능

3. **삭제하기**
   - 삭제 버튼(🗑️) 클릭
   - 확인 후 삭제

## 📱 모바일에서 사용하기

### 접속 방법

1. **같은 네트워크에서 접속** (로컬 개발)
   - PC의 IP 주소 확인:
     ```bash
     # Windows PowerShell
     ipconfig
     # IPv4 주소 확인 (예: 192.168.0.100)
     ```
   - 모바일 브라우저에서 `http://192.168.0.100:3000` 접속

2. **프로덕션 배포 후 접속** (권장)
   - 배포된 URL로 접속 (예: `https://clipbridge.vercel.app`)

### 모바일에서 사용하기

1. **저장된 클립 확인**
   - PC에서 저장한 클립이 실시간으로 동기화되어 표시됨
   - 최신 클립이 맨 위에 표시됨

2. **클립 복사**
   - 클립 카드를 터치하면 클립보드에 복사됨
   - "복사되었습니다" 메시지 표시

3. **클립 공유**
   - 공유 버튼 클릭
   - 카카오톡, 문자, 이메일 등으로 공유 가능

## 🎯 사용 팁

### 효율적인 사용법

1. **즉시 저장 활용**
   - 중요한 텍스트는 "즉시 저장" 버튼 사용
   - 실수 방지를 위해 10초 대기 활용

2. **검색 및 필터링**
   - 많은 클립이 있을 경우 브라우저의 `Ctrl+F`로 검색 가능
   - (향후 기능 추가 예정)

3. **정기적인 정리**
   - 불필요한 클립은 삭제하여 목록 관리

### 주의사항

⚠️ **보안 관련**
- 비밀번호, 신용카드 번호 등 민감한 정보는 10초 지연을 활용하여 실수 방지
- 공용 PC에서는 사용 후 데이터 삭제 권장

⚠️ **클립보드 권한**
- HTTPS 환경에서만 자동 감지 작동 (로컬은 localhost에서 가능)
- 일부 브라우저에서는 수동으로 권한 허용 필요

## 🌐 프로덕션 배포 (선택사항)

### Vercel에 배포하기 (권장)

1. **GitHub에 코드 푸시**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin your-repo-url
   git push -u origin main
   ```

2. **Vercel 배포**
   - [Vercel](https://vercel.com/) 접속
   - "New Project" 클릭
   - GitHub 저장소 선택
   - 환경 변수 설정:
     - `NEXT_PUBLIC_SUPABASE_URL`
     - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - "Deploy" 클릭

3. **배포 완료**
   - 배포된 URL로 접속 가능
   - 모바일에서도 접속 가능

### 다른 플랫폼에 배포

- **Netlify**: Vercel과 유사한 과정
- **Railway**: Docker 기반 배포
- **자체 서버**: `pnpm build` 후 `pnpm start`

## 🔧 문제 해결

### 클립보드가 감지되지 않음

1. **권한 확인**
   - 브라우저 설정에서 클립보드 권한 확인
   - 주소창 왼쪽 자물쇠 아이콘 클릭

2. **HTTPS 확인**
   - 프로덕션에서는 HTTPS 필수
   - 로컬 개발은 localhost에서 작동

3. **브라우저 호환성**
   - Chrome, Edge, Firefox 최신 버전 권장
   - Safari는 제한적 지원

### 데이터가 동기화되지 않음

1. **Supabase 연결 확인**
   - 브라우저 콘솔(F12)에서 오류 확인
   - `.env.local` 파일의 값 확인

2. **네트워크 확인**
   - 인터넷 연결 상태 확인
   - Supabase 프로젝트가 활성 상태인지 확인

### 저장이 안 됨

1. **Supabase 테이블 확인**
   - Supabase 대시보드 > Table Editor에서 `clips` 테이블 확인
   - RLS 정책 확인

2. **콘솔 오류 확인**
   - 브라우저 콘솔(F12)에서 오류 메시지 확인
   - 네트워크 탭에서 API 요청 확인

## 📊 사용 통계 확인

Supabase 대시보드에서 데이터 확인:
1. Supabase 대시보드 > **Table Editor** > `clips` 테이블
2. 저장된 모든 클립 확인 가능
3. 사용자별, 날짜별 필터링 가능
